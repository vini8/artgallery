Client and Business Goals

A big giant client operating a high user-oriented Scholarship management platform was in a trouble with website loading speed, performance, and server problems. He was looking to scale up the web application hosted over Heroku, as high-performance websites enable high return visits, low bounce rate, higher conversions, engagement, and most importantly better user experience.

Client achievement

High performance and fast loading speed improved the user experience over the scholarship management platform.

Improved the traffic flow.

Enhanced brand visibility and brand awareness.

Improved website loading speed.

Solution

Oaktree software specialized team thoroughly diagnosed the website complications, performance and performed several optimizations on a specific part of the website.

Postgres Optimization

Removed unused index

  • Indexes use up space, slow down data modification, and prevent HOT updates. Thus our Heroku team experts identified the unnecessary indexes and flawlessly took off the unused index.

Create Index in most used columns in the query

  • Postgres can create multi-column indexes; however, it is critical to understand when this is appropriate. Our developers have cautiously figured out when it is appropriate to create multi-column indexes and created indexed in every column and covered the query conditions.

Identify and fix long-running queries

  • The most common cause of database performance issues is under-optimized queries, which place significant strain on the database system. We identified these queries by looking at the expensive queries view of the database.
Code Optimization

Optimization at Code level

  • Modified the code so it consumes less memory and improved the platform code quality and efficiency.

Updated the Ruby and Rails supported versions

  • Upgraded the Ruby and Rails versions with a new version for high performance and overall reliability.

Multi-threaded web server

  • Deployed multi-thread server for enhancing the concurrency as each thread or process needs a different connection to the database.
Heroku Optimization

Use Connection Pooling

  • Integrated connection polling of Heroku Postgres that assists applications in making efficient use of database connections in order to avoid connection limits and Out of Memory errors.

pgbouncer configuration

  • Users of Heroku Postgres may reach the connection limits of their database plan. As a result, connection requests are queued, which can degrade performance and reliability. PgBouncer's connection pooling can solve this by sharing database server connections among clients.

Optimized Dyno Usage

  • Concurrency is approached differently in different languages and platforms. Effectively the application with diverse dyno types pays specific attention to memory usage and concurrency.
